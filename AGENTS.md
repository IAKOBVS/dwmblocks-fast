# dwmblocks-fast

C statusbar for window managers. Fork of dwmblocks.

## Build & test

```sh
make              # build
make check        # compile & run test (compiles with -DTEST=1)
sudo make install # strip + setcap + install to /usr/local/bin
make clean
```

- `make disable-cuda`, `make disable-x11`, `make disable-alsa` â€” toggle features by commenting out defines in `config.h` and `config.mk`.
- First build auto-copies `config.def.h` â†’ `config.h`, `config.def.mk` â†’ `config.mk`, `blocks.def.h` â†’ `blocks.h`. Edit these to configure.
- `cpu-temp-file.generated.h` is generated by `./getcpufile` (auto-run by make).

## Architecture

- Blocks run on a tick loop (`INTERVAL_UPDATE = 1s`). Each block has an `interval` (seconds between runs; `0` = signal-only).
- Block order in `g_blocks[]` determines print order after interval-sorting. `qsort` reorders by interval then signal; `internal_tostatus_idx` preserves original layout.
- **OBS order constraint**: `b_write_obs_on` must precede `b_write_obs_recording` in `g_blocks[]`.
- Signal-triggered updates: define `SIG_*` in `config.h`, set `.signal` in the block, then `pkill -RTMIN+<SIG> dwmblocks-fast`.
- Signal handler uses `sigfillset(&sa.sa_mask)` (`g_sigaction`), blocking **all** signals during handler execution. No nesting. If multiple signals are pending after `pselect` unblocks, they're delivered sequentially and `g_signal` is last-one-wins (overwritten by the final handler). This is intentional.
- Sysfs temp paths: must start with `/sys/devices/platform`, not `/sys/class` (the `path_sysfs_resolve` rewriter depends on this pattern).

## Build quirks

- `.POSIX:` make. No GNU-make-isms.
- `-fanalyzer` + `-Wpedantic -pedantic -Wall -Wextra -Wuninitialized -Wshadow -Warray-bounds -Wnull-dereference -Wformat -Wunused -Wwrite-strings`
- LTO (`-flto`) and `-march=native` enabled by default.
- Install uses `rsync -parc` if available, falls back to `cp -paf`.
- Binary gets `cap_dac_read_search+ep` via `setcap` after build and install (needed for `/sys/class/powercap/.../energy_uj`).
- `updatesig` reads `SIG_*` defines from `config.h` and prepends them as shell vars to each `scripts/` file when generating `bin/`.

## Running

```sh
dwmblocks-fast                        # X11 (sets WM_NAME on root window)
dwmblocks-fast -p                     # print to stdout (for wms that read stdin)
```

## Code conventions

- `#include "config.h"` must be first in `dwmblocks-fast.c`.
- No printf/scanf-family calls (runtime overhead). Use custom `u_utoa_p()`, `u_stpcpy()`, etc. from `utils.h`.
- Error handling via `DIE(return -1)` macro (prints `errno`, then `assert(0)`).
- `DBG(x)` macros for debug prints (enable with `-DDEBUG`).
- `#ifdef HAVE_PROCFS` / `HAVE_SYSFS` / `HAVE_POWERCAP` guard Linux-specific blocks.
- `_GNU_SOURCE` is expected for `mempcpy`, `stpcpy`, `memmem`.
